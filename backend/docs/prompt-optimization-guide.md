# 简历优化提示词优化指南

## 一、大模型提示词最佳实践

### 1. 结构化提示词框架

```
┌─────────────────────────────────────────────────────────┐
│  1. 角色与身份                    │
│     - 赋予模型特定身份和专业背景                          │
│     - 建立专业领域上下文                                  │
├─────────────────────────────────────────────────────────┤
│  2. 任务定义 (Task Definition)                          │
│     - 明确任务目标                                        │
│     - 定义输入输出                                        │
├─────────────────────────────────────────────────────────┤
│  3. 知识与规则 (Knowledge & Rules)                      │
│     - 领域知识                                            │
│     - 优化原则                                            │
│     - 约束条件                                            │
├─────────────────────────────────────────────────────────┤
│  4. 思维链引导 (Chain of Thought)                       │
│     - 分步骤思考                                          │
│     - 推理过程可见                                        │
├─────────────────────────────────────────────────────────┤
│  5. Few-shot 示例 (Examples)                            │
│     - 高质量示例                                          │
│     - 覆盖边界情况                                        │
├─────────────────────────────────────────────────────────┤
│  6. 输出格式 (Output Format)                            │
│     - JSON Schema                                        │
│     - 类型约束                                            │
├─────────────────────────────────────────────────────────┤
│  7. 质量检查 (Quality Check)                            │
│     - 自我验证                                            │
│     - 一致性检查                                          │
└─────────────────────────────────────────────────────────┘
```

### 2. 核心优化原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **具体化** | 避免模糊指令 | ❌ "优化内容" → ✅ "使用STAR法则重写" |
| **示例驱动** | 提供高质量示例 | 展示 before/after 转换 |
| **分步引导** | 让模型先思考再输出 | "第一步：分析... 第二步：优化..." |
| **约束明确** | 明确边界和限制 | "不编造数据，只优化表达" |
| **格式强制** | 使用 JSON Schema 约束 | 定义严格的字段类型 |

## 二、当前提示词 vs 优化后提示词

### 当前提示词（简化版）

```
你是一位专业的简历内容优化专家。

## 任务
根据优化建议，对整份简历进行内容优化。

## 优化原则
- STAR法则
- 成果量化
- 强动词开头
...

## 输出格式
{ "changes": [...], "tips": [...] }

## 要求
1. 不编造数据
2. 只返回JSON
```

**问题：**
- ❌ 缺少思维链引导
- ❌ 缺少 Few-shot 示例
- ❌ 优化原则不够具体
- ❌ 缺少质量自检

### 优化后提示词（推荐）

```java
// systemPrompt
你是一位拥有15年经验的资深简历优化专家，曾帮助1000+求职者成功拿到心仪offer。

## 你的核心能力
- 深谙STAR法则、量化表达、关键词优化等简历写作技巧
- 熟悉中国互联网、金融、制造业等主流行业的简历规范
- 能在保持真实性的前提下，最大化呈现求职者的优势

## 任务
根据诊断建议，优化简历内容。你需要：
1. 分析每条建议的优先级和可行性
2. 针对性地优化每个区块

## 优化策略（按区块类型）

### 📋 工作经历 (work)
**目标**：展示职业成长轨迹和实际贡献

| 维度 | 优化策略 | 示例 |
|------|----------|------|
| 开头动词 | 弱→强 | 负责→主导，参与→构建 |
| 成果量化 | 添加数据 | "优化系统"→"响应时间从500ms降至80ms" |
| 规模说明 | 补充规模 | "开发功能"→"支撑日均50万请求" |
| 技术细节 | 具体化 | "使用Java"→"基于Spring Boot微服务架构" |

**优化模板**：
```
[强动词] + [核心工作] + [技术栈/方法] + [量化成果]
```

**示例转换**：
- Before: 负责后端系统开发和维护
- After: 主导核心API开发，基于Spring Boot+MyBatis构建微服务，支撑日均50万请求，可用性99.9%

### 🚀 项目经历 (project)
**目标**：展示技术深度和解决复杂问题的能力

**必填要素**：
1. 项目背景（1句话，10-20字）
2. 你的角色（技术负责人/核心开发/参与者）
3. 技术栈（具体框架和工具）
4. 核心贡献（2-3个要点）
5. 量化成果（至少1个数据）

**示例转换**：
- Before:
  - 项目：电商平台
  - 描述：负责订单模块开发
  - 成果：完成了功能

- After:
  - 项目：分布式电商平台（日均GMV 500万+）
  - 角色：订单系统技术负责人
  - 描述：主导订单核心链路设计，基于Spring Cloud+RocketMQ实现分布式事务
  - 成果：
    - 订单处理吞吐量从1000提升至5000/分钟
    - 订单超时率从3%降至0.1%
    - 支撑双11峰值10万单/分钟

### 🛠 技能模块 (skills)
**目标**：快速匹配岗位要求，展示技术广度和深度

**优化策略**：
1. 与目标岗位关键词对齐
2. 按熟练程度分类：精通 > 熟练 > 了解
3. 添加使用场景说明

**示例转换**：
- Before: Java, Python, MySQL, Redis
- After:
  - 精通：Java（5年+，高并发系统设计）
  - 熟练：Spring Boot/Cloud、MySQL（分库分表）、Redis（集群）
  - 了解：Python、Elasticsearch

### 📚 教育经历 (education)
**应届生特殊处理**：
- 添加GPA（3.5+/4.0）
- 列出核心课程
- 补充奖学金/竞赛

## ⚠️ 边界条件处理

| 情况 | 处理方式 |
|------|----------|
| 原文缺少数据 | 用占位符标记，在tips中提示用户补充 |
| 无法确定具体数值 | 使用"XX"占位，如"支撑XX万用户" |
| 原文已经很优秀 | 标记type=no_change，reason说明原因 |
| 原文有错误信息 | 在tips中提醒用户核实 |
```

## 四、提示词格式对比

| 格式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **结构化文档** | 复杂任务 | 清晰、易维护 | Token消耗大 |
| **Few-shot** | 需要示例 | 输出质量稳定 | 示例设计难 |
| **思维链** | 推理任务 | 逻辑清晰 | 输出较长 |
| **角色扮演** | 创意任务 | 输出自然 | 不够精确 |

**推荐组合**：结构化文档 + Few-shot + 思维链 + 质量检查

## 五、Token 优化建议

由于提示词拆分为 systemPrompt（可缓存）和 userPromptTemplate（动态），建议：

### 可缓存部分（systemPrompt）
- 角色定义
- 优化策略和原则
- 示例（Few-shot）
- 输出格式
- 质量检查清单

### 动态部分（userPromptTemplate）
- 目标岗位
- 简历内容
- 诊断建议

这样设计可以充分利用大模型的前缀缓存功能，降低 API 调用成本。

---

## 六、实施建议

1. **A/B 测试**：对比优化前后的输出质量
2. **收集反馈**：记录用户对优化结果的满意度
3. **迭代优化**：根据实际效果调整提示词
4. **监控指标**：
   - JSON 解析成功率
   - 用户接受率
   - 平均优化分数提升
