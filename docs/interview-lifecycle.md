# 面试生命周期管理 - 需求文档

> 文档版本：1.0
> 创建日期：2026-03-30
> 作者：Azir

---

## 一、需求背景

LandIt 是一款智能求职助手，目前已有简历管理、AI模拟面试、面试复盘等功能。但缺少**真实面试的全流程管理能力**，用户无法系统化地管理从准备到复盘的完整面试过程。

本需求旨在将现有分散的面试功能（模拟面试、复盘）整合为统一的"面试中心"，并新增真实面试生命周期管理能力，实现：**准备 → 参加面试 → 结果跟踪 → 复盘总结** 的全流程覆盖。

---

## 二、核心概念

### 2.1 面试（Interview）

面试分为两种类型，统一管理：

| 类型 | 说明 | 来源 |
|------|------|------|
| **真实面试**（Real） | 用户真实的求职面试 | 手动录入 |
| **模拟面试**（Mock） | AI模拟的练习面试 | 系统生成 |

### 2.2 面试生命周期（真实面试）

```
创建面试 → 面试准备 → 参加面试 → 结果跟踪 → 复盘总结
   |           |           |           |           |
   |      公司调研      记录实况    多轮状态    手动记录
   |      JD分析       录音/笔记    流转管理    AI辅助分析
   |      模拟训练
   |      准备清单
   v
(模拟面试走现有流程：开始 → 答题 → 结束 → 自动复盘)
```

### 2.3 多轮面试（Round）

一次真实面试可能包含多轮，每轮独立跟踪：

- **预设轮次类型**：技术一面、技术二面、HR面、总监面、CTO面、终面
- **用户自定义**：用户可添加自定义名称的轮次
- 每轮独立状态：待面试 → 进行中 → 已通过 → 未通过 → 待定 → 已取消

## 三、前端结构

### 3.1 导航改造

**现有**：4个Tab（工作台、简历管理、面试试演[NEW]、面试复盘）

**改造后**：4个Tab（工作台、简历管理、**面试中心**、个人中心）

- 合并"面试试演"和"面试复盘"为"面试中心"
- 面试复盘不再作为独立Tab，而是面试中心的子功能

### 3.2 面试中心页面结构

```
面试中心（/interview-center）
├── 面试列表页（默认视图）
│   ├── Tab: 全部 | 真实面试 | 模拟面试
│   ├── 面试卡片列表（按时间倒序）
│   └── + 新建面试按钮
│
├── 面试详情页（/interview-center/:id）
│   ├── 面试基本信息（公司、岗位、日期等）
│   ├── 多轮面试时间线
│   ├── 面试准备区
│   │   ├── 公司调研资料
│   │   ├── JD分析结果
│   │   ├── 准备事项清单（todo list）
│   │   └── 针对性模拟训练入口
│   ├── 面试记录区
│   │   ├── 手动记录面试感受/笔记
│   │   └── 上传录音/文件（可选）
│   ├── 结果跟踪区
│   │   └── 各轮面试状态卡片
│   └── 复盘总结区
│       ├── 手动复盘记录
│       └── AI辅助分析
│
├── 模拟面试页（/interview-center/mock）
│   └── 保留现有模拟面试功能（开始/进行中/录音回放）
│
└── 复盘历史页（/interview-center/reviews）
    └── 保留现有复盘列表和详情功能
```

### 3.3 页面交互流程

#### 创建真实面试
```
点击"新建面试" → 填写基本信息表单
  ├── 公司名称（必填）
  ├── 目标岗位（必填）
  ├── 面试日期（必填）
  ├── 粘贴JD（选填，填了可触发AI分析）
  ├── 预设轮次（多选：技术一面/技术二面/HR面等）
  ├── 自定义轮次（可添加）
  └── 备注（选填）
→ 创建成功，进入面试详情页（准备阶段）
```

#### 面试准备流程
```
进入面试详情 → 准备阶段
  ├── [公司调研] → AI自动分析公司信息（基于公司名称+JD）
  ├── [JD分析] → AI提取关键词、技能要求、匹配度分析
  ├── [准备清单] → 自动生成 + 手动增删改的 todo list
  └── [模拟训练] → 基于JD生成针对性模拟面试题，跳转模拟面试
```

#### 结果跟踪流程
```
进入面试详情 → 结果跟踪区
  ├── 时间线展示各轮面试
  ├── 每轮可独立更新状态（待面试/通过/未通过/待定/取消）
  ├── 可追加轮次（面试过程中又加了一轮）
  └── 所有轮次结束后，面试整体状态自动更新
```

#### 复盘流程
```
面试结束后 → 复盘总结区
  ├── 手动记录
  │   ├── 面试感受（文字）
  │   ├── 表现自评（1-5星）
  │   └── 经验教训
  └── AI辅助分析
      ├── 基于JD + 面试记录分析表现
      ├── 生成改进建议
      └── 推荐后续学习/训练方向
```

## 四、功能详细描述

### 4.1 面试中心首页

**功能**：统一展示所有面试（真实+模拟），支持筛选和快速操作

**需求点**：
1. 顶部Tab切换：全部 / 真实面试 / 模拟面试
2. 面试卡片展示：公司名、岗位、日期、当前阶段/状态、轮次进度
3. 右上角"新建面试"按钮（创建真实面试）
4. "模拟面试"快捷入口按钮（跳转现有模拟面试流程）
5. 卡片点击进入面试详情
6. 模拟面试卡片显示已有功能（开始面试/继续面试/查看复盘）

### 4.2 创建真实面试

**功能**：手动录入真实面试信息

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 公司名称 | 文本 | 是 | 面试公司名称 |
| 目标岗位 | 文本 | 是 | 如"Java高级工程师" |
| 面试日期 | 日期 | 是 | 第一轮面试日期 |
| JD内容 | 富文本 | 否 | 粘贴职位描述，填了可触发AI分析 |
| 面试轮次 | 多选+自定义 | 否 | 预设类型 + 用户自定义 |
| 备注 | 文本 | 否 | 补充说明 |

**预设轮次类型**：
- 技术一面（Technical Round 1）
- 技术二面（Technical Round 2）
- HR面（HR Interview）
- 总监面（Director Interview）
- CTO/VP面（Executive Interview）
- 终面（Final Interview）

### 4.3 面试准备

#### 4.3.1 公司调研资料

**功能**：AI基于公司名称和JD自动生成公司调研资料

**内容包含**：
- 公司基本信息（行业、规模、业务方向）
- 技术栈推测（基于JD）
- 公司文化特点
- 近期动态（如有）

**交互**：
- 创建面试时若填了JD，自动触发分析
- 未填JD可后续补充触发
- 支持手动编辑/补充调研内容

**降级方案**：

| 情况 | 处理方式 |
|------|---------|
| 公司名称不完整（如"阿里"） | AI 尽量推测，结果标注"仅供参考" |
| 公司信息获取失败 | 返回基础结构 + 用户手动填写 |
| JD内容为空 | 仅基于公司名称推测，准确度降低 |

> ⚠️ **重要提示**：公司调研结果基于 AI 推测，**不依赖外部 API**，准确性取决于输入信息完整度。前端需展示"AI 推测结果仅供参考，请根据实际情况修正"提示。

#### 4.3.2 JD分析

**功能**：AI分析职位描述，提取关键信息

**分析维度**：
- 核心技能要求（必须掌握的）
- 加分技能（nice to have）
- 关键词提取
- 岗位级别判断（初级/中级/高级）
- 与用户简历的匹配度分析（关联简历模块）

**交互**：
- 粘贴JD后点击"分析"触发AI工作流
- 分析结果可展开/折叠查看
- 支持重新分析

#### 4.3.3 准备事项清单

**功能**：自动生成 + 手动管理的面试准备todo list

**自动生成逻辑**：
- 基于JD分析结果，AI生成建议的准备事项
- 如："复习XX框架原理"、"准备项目经验描述"、"了解公司XX产品"

**优先级计算规则**：

| 优先级 | 计算依据 | 示例 |
|--------|---------|------|
| 高（high） | JD核心技能 + 简历技能差距 | "复习Spring Boot原理（JD要求核心技能）" |
| 中（medium） | JD加分技能 + 公司调研建议 | "了解公司主要产品线" |
| 低（low） | 通用面试准备 | "准备自我介绍" |

> 💡 优先级 = 核心技能匹配度 × 技能差距权重，系统自动计算，用户可手动调整

**交互**：
- 每项可勾选完成/取消完成
- 可手动添加新事项
- 可删除事项
- 可编辑事项内容
- 显示完成进度（3/8 已完成）
- 按优先级排序展示（高→中→低）

#### 4.3.4 针对性模拟训练

**功能**：基于面试岗位和JD，生成针对性的模拟面试

**实现方式**：
- 点击"开始模拟训练" → 基于JD关键词和岗位生成模拟面试题
- 复用现有模拟面试的答题、评分、录音等流程
- 模拟训练结果在"准备区"显示入口

**数据关联说明**：

| 关联类型 | 说明 |
|---------|------|
| 显示关联 | 在真实面试详情页展示"已完成 X 次模拟训练"入口 |
| 跳转关联 | 点击入口跳转到模拟面试详情/复盘页（不内嵌） |
| 数据独立 | 模拟面试数据存储在 `t_interview_session`，通过 `session_id` 关联 |

> 💡 **设计决策**：模拟面试和真实面试数据独立存储，通过前端路由参数实现逻辑关联，不建立强外键约束。这样设计的好处是：
> - 模拟面试可以独立存在（非针对性训练）
> - 数据结构解耦，便于后续扩展

### 4.4 参加面试（记录）

**功能**：记录面试过程中的实际情况

**记录方式**：
- 手动文字记录面试感受、遇到的题目、自己的表现
- 可上传录音文件（可选，非必须）
- 每轮面试独立记录

**记录字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 面试笔记 | 富文本 | 记录面试过程中遇到的问题、自己的回答要点等 |
| 面试感受 | 文本 | 整体感受描述 |
| 自我评价 | 1-5星 | 对自己表现的主观评价 |
| 附件 | 文件 | 上传录音或其他文件（可选） |

**文件上传限制**：

| 类型 | 支持格式 | 大小限制 | 说明 |
|------|---------|---------|------|
| 音频 | mp3, wav, m4a, ogg | ≤ 50MB | 面试录音 |
| 文档 | pdf, doc, docx, txt | ≤ 20MB | 面试笔记/资料 |
| 图片 | jpg, png | ≤ 10MB | 截图/手写笔记 |

> ⚠️ 上传的文件存储在服务端，后续可考虑增加云存储支持

### 4.5 结果跟踪

**功能**：多轮面试的状态管理和进度跟踪

**轮次状态流转**：
```
待面试 → 进行中 → 已通过
                  → 未通过
                  → 待定（等待结果）
         → 已取消
```

**功能点**：
1. 时间线视图展示所有轮次
2. 每轮可独立更新状态和结果
3. 可追加新轮次（面试过程中临时加面）
4. 可编辑/删除轮次
5. 整体面试状态根据轮次结果自动推断（见下方规则表）

**状态联动规则表**：

| 条件 | 整体状态 | 说明 |
|------|---------|------|
| 所有轮次"已通过" | completed (passed) | 全部通过 |
| 任一轮次"未通过" | completed (failed) | 一票否决 |
| 任一轮次"待定"且无"未通过" | in_progress | 等待结果 |
| 仍有"待面试"或"进行中" | in_progress | 面试进行中 |
| 删除所有轮次 | preparing | 回到准备阶段 |
| 取消所有轮次 | cancelled | 面试取消 |

> 💡 **状态联动触发时机**：
> - 用户更新轮次状态时自动计算
> - 用户删除轮次时自动重新计算
> - 不需要手动更新整体状态

**终态说明**：
- `passed`、`failed`、`cancelled` 为终态，进入后不可逆转
- 如需重新开始，需创建新的面试记录

### 4.6 复盘总结

**功能**：面试后的总结和改进分析

#### 手动复盘
- 面试整体感受（文字）
- 好的方面 / 不足之处
- 经验教训
- 下次改进方向

#### AI辅助分析
- 基于面试记录（笔记 + JD + 结果）进行AI分析
- 分析维度：
  - 表现评价（基于记录内容的智能分析）
  - 改进建议（具体可执行的改进项）
  - 推荐训练方向（建议做哪些模拟面试练习）
  - 技能提升建议（哪些技能需要加强）
- AI分析通过全新工作流实现

## 五、数据模型设计（概要）

### 5.1 新增/修改的表

#### t_interview（改造现有表）

现有表已有基础字段，需扩展支持真实面试：

| 新增字段 | 类型 | 说明 |
|----------|------|------|
| interview_source | VARCHAR(20) | 来源：real（真实）/ mock（模拟），默认mock |
| jd_content | TEXT | JD内容 |
| status | VARCHAR(20) | 扩展状态枚举：preparing/in_progress/completed/cancelled |
| overall_result | VARCHAR(20) | 最终结果：passed/failed/pending |
| notes | TEXT | 备注 |
| company_research | TEXT | 公司调研结果（JSON） |
| jd_analysis | TEXT | JD分析结果（JSON） |

#### t_interview_round（新增表）

面试轮次表，管理多轮面试：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) | 主键 |
| interview_id | VARCHAR(64) | 关联面试ID |
| round_type | VARCHAR(30) | 轮次类型：technical_1/technical_2/hr/director/cto/final/custom |
| round_name | VARCHAR(50) | 轮次名称（自定义时使用） |
| round_order | INTEGER | 轮次顺序 |
| status | VARCHAR(20) | 状态：pending/in_progress/passed/failed/pending_result/cancelled |
| scheduled_date | DATETIME | 预定日期 |
| actual_date | DATETIME | 实际日期 |
| notes | TEXT | 面试笔记 |
| self_rating | INTEGER | 自我评价（1-5） |
| result_note | TEXT | 结果说明 |
| 通用字段 | - | created_at, updated_at, deleted |

#### t_interview_preparation（新增表）

面试准备清单表：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) | 主键 |
| interview_id | VARCHAR(64) | 关联面试ID |
| item_type | VARCHAR(20) | 类型：company_research/jd_analysis/todo/manual |
| title | VARCHAR(200) | 标题 |
| content | TEXT | 内容 |
| completed | BOOLEAN | 是否完成 |
| source | VARCHAR(20) | 来源：ai_generated/manual |
| sort_order | INTEGER | 排序 |
| 通用字段 | - | created_at, updated_at, deleted |

#### t_interview_review_note（新增表）

面试复盘笔记表（区别于现有的t_interview_review自动复盘）：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) | 主键 |
| interview_id | VARCHAR(64) | 关联面试ID |
| type | VARCHAR(20) | 类型：manual/ai_analysis |
| high_points | TEXT | 好的方面 |
| weak_points | TEXT | 不足之处 |
| lessons_learned | TEXT | 经验教训 |
| suggestions | TEXT | AI建议（JSON） |
| overall_feeling | TEXT | 整体感受 |
| 通用字段 | - | created_at, updated_at, deleted |

### 5.2 复用的现有表

| 表 | 说明 |
|----|------|
| t_interview_session | 模拟面试会话，继续使用 |
| t_interview_question | 题库，模拟面试和针对性训练都用 |
| t_conversation | 面试对话，模拟面试用 |
| t_interview_review | AI复盘报告，模拟面试和真实面试AI分析都用 |

## 六、API 设计（概要）

### 6.1 真实面试管理 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /interview-center | 创建真实面试 |
| GET | /interview-center | 获取面试列表（支持 type=real/mock/all 筛选） |
| GET | /interview-center/{id} | 获取面试详情（含轮次、准备、复盘） |
| PUT | /interview-center/{id} | 更新面试基本信息 |
| DELETE | /interview-center/{id} | 删除面试 |
| PATCH | /interview-center/{id}/status | 更新面试整体状态 |

### 6.2 面试轮次 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /interview-center/{id}/rounds | 新增面试轮次 |
| PUT | /interview-center/{id}/rounds/{roundId} | 更新轮次信息 |
| PATCH | /interview-center/{id}/rounds/{roundId}/status | 更新轮次状态 |
| DELETE | /interview-center/{id}/rounds/{roundId} | 删除轮次 |
| PUT | /interview-center/{id}/rounds/reorder | 调整轮次顺序 |

### 6.3 面试准备 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /interview-center/{id}/prepare/research | 触发AI公司调研 |
| POST | /interview-center/{id}/prepare/jd-analysis | 触发AI JD分析 |
| GET | /interview-center/{id}/prepare/todos | 获取准备清单 |
| POST | /interview-center/{id}/prepare/todos/generate | AI生成准备事项 |
| POST | /interview-center/{id}/prepare/todos | 手动添加准备事项 |
| PUT | /interview-center/{id}/prepare/todos/{todoId} | 更新准备事项 |
| DELETE | /interview-center/{id}/prepare/todos/{todoId} | 删除准备事项 |

### 6.4 面试复盘 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /interview-center/{id}/review | 获取复盘笔记 |
| POST | /interview-center/{id}/review | 保存手动复盘 |
| POST | /interview-center/{id}/review/ai-analysis | 触发AI辅助分析 |

### 6.5 针对性模拟训练 API

## 七、AI 工作流设计（概要）

### 7.1 公司调研工作流（InterviewResearchGraph）

```
START → CollectCompanyInfo → AnalyzeTechStack → GenerateReport → END
```

- 输入：公司名称 + JD内容
- 输出：公司调研报告（JSON）

### 7.2 JD分析工作流（JDAnalysisGraph）

```
START → ExtractRequirements → AnalyzeKeywords → EvaluateMatch → GenerateSuggestions → END
```

- 输入：JD内容 + 用户简历（可选）
- 输出：技能要求、关键词、匹配度、建议

### 7.3 面试复盘AI分析工作流（InterviewReviewAnalysisGraph）

```
START → AnalyzeNotes → EvaluatePerformance → GenerateImprovements → RecommendTraining → END
```

- 输入：面试记录 + JD + 轮次结果
- 输出：表现分析、改进建议、推荐训练方向

### 7.4 准备事项生成工作流（PreparationTodoGraph）

```
START → AnalyzeJD → GenerateTodos → Prioritize → END
```

- 输入：JD分析结果
- 输出：优先级排序的准备事项列表

## 九、与现有功能的整合点

| 现有功能 | 整合方式 |
|----------|----------|
| 模拟面试（InterviewSession） | 保留，作为面试中心的子功能，同时支持从真实面试准备中发起针对性训练 |
| 面试复盘（Review） | 保留，模拟面试自动复盘 + 真实面试手动/AI复盘，统一入口 |
| 面试题库（Question） | 复用，针对性模拟训练从题库中按标签筛选相关题目 |
| 语音面试 | 保留，模拟面试继续支持语音模式 |
| 简历模块 | 联动，JD分析时可对比用户简历，给出匹配度 |
| AI聊天（Chat） | 可联动，用户可通过AI聊天询问面试准备相关问题 |
